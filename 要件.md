DataDeck for VS Code: 要件定義書
1. プロジェクト概要
データサイエンティストが Jupyter Notebook や marimo で分析を行う際、試行錯誤の過程で生成される膨大なグラフ、表、考察（メモ）を、スクロールの手間なく一箇所に集約・整理・再利用できるようにする VS Code 拡張機能。

2. ターゲットユーザー
VS Code の Native Notebook インターフェース、または marimo を利用するデータサイエンティスト。

実験管理ツール（WandB等）を使うほどではないが、ローカルでの試行錯誤を効率化したい開発者。

3. 主要機能要件
3.1 成果物のキャプチャ (Clip & Store)
クイック保存: Notebook のセル出力（画像、HTML、DataFrameプレビュー）をワンクリック、またはショートカットでサイドバーの「デッキ」に保存。

タグ/メモ付与: 保存時に、その出力に対する短い考察やメモをその場で入力可能。

コードスナップショット: 出力時のセルコードを自動的に紐付けて保存。

3.2 サイドバー・ダッシュボード (The Deck)
固定表示 (Pinning): Notebook をスクロールしても、重要なグラフをサイドバーに常駐させ、いつでも比較可能にする。

タイムライン表示: 保存した順にカード形式でリスト表示。

ソースジャンプ: 保存されたカードをクリックすると、その出力を生成した Notebook の該当セルへ即座にスクロール・移動。

3.3 整理・検索機能
ドラッグ＆ドロップ: サイドバー内での表示順序の入れ替え。

フィルタリング: ファイル名、タグ、保存日時、出力タイプ（画像/表）による絞り込み。

3.4 エクスポート (Reporting)
Markdown出力: 選択した成果物とメモをまとめ、画像付きのレポート下書き（.md）として出力。

4. 非機能要件
パフォーマンス: Notebook の動作を妨げないバックグラウンド処理。

ポータビリティ: 保存データはプロジェクトディレクトリ内（.vscode/datadeck/ 等）に保存し、共有や Git 管理を可能にする（オプションで GitIgnore 推奨）。

互換性: * VS Code Native Notebook API への対応。

marimo のセル出力・スクリプト構造への対応。

5. 画面構成（UI）案
Activity Bar Icon: DataDeck 専用のアイコン。

Primary Side Bar: - Pinned セクション（常に見ていたいグラフ）。

Recent Clips セクション（時系列の保存履歴）。

Floating Action Button (Optional): セル出力の右肩に表示される「Add to Deck」ボタン。

6. 技術スタック案
Language: TypeScript

Framework: VS Code Extension API

Frontend (Side Panel): React または Vue.js (VS Code Webview API)

Storage: Local File System (JSON + Image files)
