# DataDeck for VS Code: 設計書

## 1. システム概要
VS Code拡張機能として、データサイエンティストの試行錯誤過程を支援する「クリップボード型」成果物管理システム。

## 4. データモデル設計

### 4.1 Clip (クリップ)
```typescript
interface Clip {
  id: string;                     // UUID v4
  timestamp: number;              // Unixタイムスタンプ(ms)
  type: 'image' | 'html' | 'dataframe' | 'text';
  title?: string;                 // ユーザー指定タイトル
  memo?: string;                  // メモ・考察
  tags: string[];                 // タグ配列
  source: {
    notebookUri: string;          // ソースNotebookのURI
    cellId?: string;              // セルID (Notebook API)
    executionCount?: number;      // 実行番号
  };
  content: {
    imagePath?: string;           // 相対パス (.vscode/datadeck/images/)
    htmlContent?: string;         // HTML文字列（データフレーム等）
    textContent?: string;         // プレーンテキスト
    mimeType?: string;            // 元のMIMEタイプ
  };
  codeSnippet?: string;           // 生成時のコード
  pinned: boolean;                // ピン留め状態
  order: number;                  // 手動並び替え用
  metadata: {
    fileSize?: number;           // 画像サイズ等
    dimensions?: {               // 画像の場合
      width: number;
      height: number;
    };
  };
}
```

### 4.2 Deck (デッキ)
```typescript
interface Deck {
  version: string;                // スキーマバージョン "1.0.0"
  lastUpdated: number;            // 最終更新日時
  clips: Clip[];                 // クリップ配列
  settings: {
    autoSave: boolean;           // 自動保存設定
    maxClips: number;            // 保持上限
    imageQuality: number;        // 画像保存品質(0-100)
  };
}
```

### 4.3 保存形式 (clips.json)
```json
{
  "version": "1.0.0",
  "lastUpdated": 1714560000000,
  "clips": [...],
  "settings": {...}
}
```

## 5. 主要コンポーネント設計

### 5.1 Extension Entry (extension.ts)
```typescript
export function activate(context: ExtensionContext) {
  // コマンド登録
  registerCommand('datadeck.clipOutput', clipOutput);
  registerCommand('datadeck.exportMarkdown', exportMarkdown);
  registerCommand('datadeck.clearDeck', clearDeck);
  
  // サイドバー登録
  const provider = new SidebarProvider(context.extensionUri);
  window.registerWebviewViewProvider('datadeck.sidebar', provider);
  
  // Notebook API初期化
  initializeNotebookAdapter(context);
}
```

### 5.2 ClipboardService
- **役割**: Notebookセル出力のキャプチャと保存
- **主要メソッド**:
  - `clipActiveCell()`: アクティブなセルをクリップ
  - `captureOutput(output, context)`: 出力データをキャプチャ
  - `saveImage(dataUrl, filename)`: 画像を保存
  - `associateCode(cell)`: コードスニペットを紐付け

### 5.3 StorageService
- **役割**: データの永続化と読み込み
- **保存先**: `${workspaceFolder}/.vscode/datadeck/`
- **主要メソッド**:
  - `loadDeck(): Promise<Deck>`
  - `saveDeck(deck: Deck): Promise<void>`
  - `saveImage(base64Data: string): Promise<string>` (パスを返す)
  - `deleteClip(clipId: string): Promise<void>`

### 5.4 SidebarProvider (Webview)
- **役割**: ReactベースのサイドバーUI
- **通信**: postMessage/message でExtensionと通信
- **表示セクション**:
  - Pinned Clips (常駐表示)
  - Recent Clips (タイムライン)
  - Search & Filter Bar

### 5.5 NotebookAdapter
- **役割**: Notebook APIの抽象化
- **対応先**:
  - VS Code Native Notebook
  - marimo (Pythonスクリプトの場合の特別処理)

## 8. 非機能要件の実現方針

### 8.1 パフォーマンス
- 画像の遅延読み込み (Lazy Loading)
- Web Workerでの画像圧縮処理
- 大量クリップ時の仮想スクロール (react-window)

### 8.2 ポータビリティ
- プロジェクトルートの `.vscode/datadeck/` に集約
- `.gitignore` への追加を推奨（設定可能）
- 相対パスで管理し、プロジェクトごと移動可能

### 8.3 互換性
- VS Code Native Notebook API (notebookDocument, cell outputs)
- marimo: `marimo.App` の解析とセル特定
- Jupyter Notebook (.ipynb) も間接的にサポート
